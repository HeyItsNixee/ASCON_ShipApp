17.10.2022 [rev 118351]
Функция GCE_StateOfSystem переименована на GCE_DefinitionState и поменяла возвращаемые
значения в соотвествии с описанием:
    Функция вернет состояние GCE_STATE_Underconstrained, если имеется хотя бы
    один геометрический объект с ненулевой степенью свободы. Состояние
    GCE_STATE_WellConstrained означает, что геометрия полностью определена, а
    другое состояние GCE_STATE_UnresolvedRedundancy означает, что в модели имеются
    нерешенные избыточные ограничения.
    Состояние GCE_STATE_OverConstrained больше не присваивается.
--------------------------------------------------------------------------------
The function GCE_StateOfSystem was renamed to GCE_DefinitionState and its resulting
values was changed accщквштп to the following description:
    The function will return #GCE_STATE_Underconstrained if there is at least
    one geometric object with nonzero degree of freedom. The GCE_STATE_WellConstrained
    state means that the geometry is fully-defined. And the other state is
    GCE_STATE_UnresolvedRedundancy that means there are unresolved redundant
    constraints (non solved overdefinition).
    The GCE_STATE_OverConstrained state is no longer assigned.
